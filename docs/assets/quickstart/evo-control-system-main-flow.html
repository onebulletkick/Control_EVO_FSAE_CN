EVO_Control_System 快速上手：主控制链路

Bus 化主数据流：VehicleStateBus -> TargetBus -> YawMomentBus -> TorqueRequestBus / AllocationBus -> ActuatorBus
来自 CarSim
S-Function
sensor_bus
➜
1
VehicleStateInput
车辆状态整理

接收 CarSim sensor_bus；提取车速、横摆角速度、侧偏、转角、轮速等。

输出：VehicleStateBus
2
ControlTarget
Generation
目标生成

根据车速、前轮转角和附着限制，计算理想横摆角速度 wr_d 与侧偏角 beta_d。

输出：TargetBus
3
YawMomentControl
横摆力矩控制

PID / MPC 对比目标与当前状态，计算附加横摆力矩 Mz。

输出：YawMomentBus
4
TorqueRequest
基础扭矩请求

把油门、制动和车速状态转成基础轮端扭矩，并处理驱动/制动平滑。

输出：TorqueRequestBus
继续进入分配与执行器限幅
5
TorqueAllocation
扭矩分配

把基础扭矩和附加横摆力矩分配到四轮；支持载荷比例法或 QP。

输出：AllocationBus
➜
6
ActuatorOutputLimiter
执行器限幅

施加电机限扭、总功率限制、扭矩变化率和最终饱和。

输出：ActuatorBus / T_to_CarSim_4w_N_m
➜
7
T 输出到 CarSim
回到 CarSim

四轮轮端扭矩回到 CarSim S-Function，形成闭环仿真。

输出：T_to_CarSim_4w_N_m
